#简单工厂
```
简单工厂不是一个标准的设计模式，但是非常常用。
定义：
    提供一个创建对象实例的功能，而无须关系其具体实现。
    被创建实例的类型可以是接口、抽象类，也可以是具体的类。

简单工厂的功能
    可以用来创建的接口、抽象类或者普通类的实例
静态工厂
    通常把简单工厂类实现成一个工具类，直接使用静态方法就可以了，也就是说简单工厂的方法通常都是静态的，所以也被称为静态工厂。
万能工厂
    一个简单工厂理论上可以用来构造任何对象，所以又称之为“万能工厂”
简单工厂创建对象的范围
    建议控制在一个独立的组件级别或者一个模块级别

简单工厂命名的建议：
    类名建议为“模块名称+Factory"
    方法名称通常为"get+接口名称"或者是“create+接口名称”
    不建议把方法名称命名为“new+接口名称”

简单工厂中方法的写法
    简单工厂方法的内部主要实现的功能是“选择合适的实现类”来创建实例对象。

选择的参数来源：
    来源于client
        注意：客户端必须知道每个参数的含义，也需要理解每个参数对应的功能处理。
        这就要求必须在一定程度上，像客户暴露一定的内部实现细节。
    来源于配置文件
        可配置的简单工厂。从配置文件读取相应配置。
    来源于系统自身，比如运行期间的某个值
    
简单工厂的优缺点：
    帮助封装
    解耦
    可能增加客户端的复杂度
    不方便拓展子工厂

简单工厂的本质是：选择实现
    实现应该是预先做好的，工厂只是选择
```
# 简单工厂的选择
```
如果想要完全封装隔离具体实现，让外部只能通过接口来操作封装体。
如果想要把对外创建对象的职责机制管理和控制，可以选用简单工厂。
```
